# Design

This section contains the canonical design and documentation guidance for Rock Hero.

- [Architecture](architecture.md)
- [Architectural Principles](architectural-principles.md)
- [Documentation Style](documentation-style.md)

Read these documents together:

- `architectural-principles.md` defines the structural rules that keep the codebase testable and
  maintainable as it grows.
- `documentation-style.md` defines the comment and Doxygen conventions for project-owned code.
